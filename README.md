## Items API (FastAPI тестовое задание)

Простое CRUD‑API на основе **FastAPI** и **Pydantic** для работы с сущностью `Item`.

### Функциональность

- **List**: получить список элементов с поддержкой фильтрации и пагинации.
- **Get**: получить один элемент по `id`.
- **Create**: создать новый элемент.
- **Update**: обновить существующий элемент (частичное/полное обновление).
- **Delete**: удалить элемент.

Данные хранятся в простом in‑memory хранилище (словарь в памяти процесса), поэтому все изменения доступны, пока запущен сервер.

### Установка и запуск (локально)

1. **Клонировать репозиторий**

```bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>.git
cd FastAPI_test
```

2. **Создать и активировать виртуальное окружение (рекомендуется)**

```bash
python -m venv .venv
.venv\Scripts\activate  # Windows
# source .venv/bin/activate  # Linux / macOS
```

3. **Установить зависимости**

```bash
pip install -r requirements.txt
```

4. **Запустить приложение**

```bash
uvicorn app.main:app --reload
```

По умолчанию приложение будет доступно по адресу: `http://127.0.0.1:8000`

### Эндпоинты

Базовый префикс для всех эндпоинтов: `/items`

- **GET `/items/`** — получить список элементов.
  - Query‑параметры:
    - `skip` (int, по умолчанию 0) — сколько элементов пропустить (пагинация).
    - `limit` (int, по умолчанию 100) — сколько элементов вернуть.
    - `name_contains` (str, опционально) — фильтр по подстроке в названии.
    - `is_active` (bool, опционально) — фильтр по признаку активности.
- **GET `/items/{item_id}`** — получить один элемент по `id`.
- **POST `/items/`** — создать новый элемент.
- **PUT `/items/{item_id}`** — обновить существующий элемент.
- **DELETE `/items/{item_id}`** — удалить элемент.

Все детали и примеры тел запросов/ответов доступны в Swagger UI.
